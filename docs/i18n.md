# Locales and Translations

## Locale model

`CmsLocale` includes:

- `code` (e.g. `en`, `nl`, `nl-BE`)
- `language`
- optional `country`
- optional `default`

Helpers normalize locale casing (`nl-be` -> `nl-BE`) and split language/country safely.

## Translated routes

`CmsRoute.translations` stores locale-to-path mapping.

Example:

```json
{
  "en": "/contact-us",
  "nl": "/contacteer-ons"
}
```

Resolver can match incoming slugs against any translation and determine canonical output for the selected locale.

## SEO alternates

Metadata helper can emit alternates:

- canonical URL
- language URLs (`hreflang` style mapping)

Use `localeToHrefLang` when your locale code and hreflang format differ.

## Locale-aware API requests

CMS client supports locale-aware requests:

- locale query parameter
- `Accept-Language` header

so data fetch behavior is explicit and environment-independent.
